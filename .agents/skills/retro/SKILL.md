name: retro
description: Retrospective — reflect on the work done, then clean up the solution
---

Ok, you found a working solution! Now that you actually know how to make it work, think:
- Can you figure out a better way to implement or structure the actual solution? First principles thinking, you can get creative for this if it makes sense.
- Is there any way you could simplify it, clean it, or make it more elegant?
- Did you write code that typically already exists in large codebases like ours and could be a duplicate of that? Feel free to explore the codebase looking for this if you think its likely.

You are free to think and explore as much as you want, but only communicate to actionable suggestions, do not show me your thought process or things that you could've done but that don't actually think should be done.

If there is nothing to do just say so. Be as terse as possible in your response. Really, only reply with good ideas and things you actually think we should implement as they would actually improve the solution.
